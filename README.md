# 시각장애인 체험 시뮬레이션

> **"빛을 잃은 보도 위, 노란 선을 따라 걷다"**

이 프로젝트는 시각장애인의 보행 환경을 직접 3D로 체험함으로써 인도 위 불법 주정차와 파손된 점자블록이 시각장애인에게 미치는 실질적인 위협을 알리고 공감대를 형성하기 위해 기획된 **3D 웹 시뮬레이터**입니다.

## 🔗 프로젝트 특징
- **웹 브라우저 즉시 실행**: 무거운 설치 없이 크롬 등 모던 웹 브라우저에서 HTML5, Three.js 기술을 기반으로 즉시 구동됩니다.
- **직관적이고 단순한 UI**: 접속 즉시 1인칭 시점의 시뮬레이션을 시작할 수 있습니다.
- **음성 안내 피드백**: 브라우저의 Web Speech API (TTS)를 통해 상황(신호등 상태 변경, 충돌, 클리어, 게임오버)에 맞는 친절한 한국어 음성 안내가 출력됩니다.

---

## 🎮 시뮬레이션 주요 기능
1. **3D 몰입형 보행 환경 (Three.js)**
   - 가운데 황색 점선과 양쪽 실선이 그려진 2차선 도로 및 횡단보도 (총 3개의 횡단보도 배치).
   - 보도의 중심을 따라 깔려 있는 황색 점자블록(유도용 선형 블록 & 대기용 점형 블록).
   - 주변 건물, 가로수, 가로등 배치로 현장감 넘치는 도시 보도 구현.

2. **시각 제한 페이딩 (암전 효과)**
   - 평소 점자블록 위 또는 횡단보도를 올바르게 걷고 있을 때는 시야가 정상적으로 보입니다.
   - 전방에 **파손된 점자블록**이 있거나, 점자블록 및 횡단보도를 **벗어나면** 화면이 약 99% 어두워지는 암전 효과(시각 제한)가 활성화됩니다.

3. **배리어 프리 음향 신호기 시스템**
   - 횡단보도 앞 기둥에 설치된 신호등 (적색 5초 ➡️ 초록 5초 ➡️ 노랑 1.5초 주기).
   - **음성 안내 버튼 상호작용**: 평소에는 신호등 색상이 흑백으로 보이다가, 신호기 기둥 근처에서 **E 키**를 누르거나 마우스로 버튼을 조준하여 클릭하면 음성 안내 방송(*"신호등 음성 안내입니다. 현재 초록불입니다."*)과 함께 신호가 컬러로 밝혀집니다. 또한 신호가 바뀔 때마다 안내 음성이 자동으로 재생됩니다.

4. **현실적인 위험 요소 및 게임 오버**
   - **무단횡단 사고**: 신호등이 **빨간불**일 때 횡단보도를 건너면 즉시 차량 충돌 게임 오버가 발생합니다.
   - **불법 주정차 충돌**: 점자블록 주위에 불법 주차된 방해 차량이 있을 때 경고 문구가 출력되며, 충돌 시 3초간 이동 불능 상태가 됩니다. 3회 누적 충돌 시 게임 오버됩니다.
   - **목적지 도달 (CLEAR)**: 마지막 신호등을 건너 목적지(녹색 표시 구역)에 무사히 도달하면 성공적으로 시뮬레이션이 클리어됩니다.

## ⌨️ 조작 방법
- **이동**: `W` `A` `S` `D` 키 (1인칭 시점 이동) / `Shift` 키를 누르면 빠르게 이동합니다.
- **시야 회전**: 마우스 움직임 (화면 클릭 시 마우스 잠금 활성화, `ESC` 키로 해제)
- **신호기 작동**: 기둥 근처에서 `E` 키 또는 기둥의 버튼 클릭
- **다시 시작**: 게임 오버 또는 클리어 시 모달창의 버튼을 누르거나 클릭하여 리셋

## 🛠️ 실행 방법 (로컬 호스팅)
웹 오디오 API 및 TTS(음성 안내) 서비스는 브라우저의 보안 정책상 파일 프로토콜 (`file:///`)에서는 일부 제한될 수 있으므로, 아래와 같이 로컬 서버를 통해 구동하는 것을 권장합니다.

1. 터미널을 열고 프로젝트 폴더 경로로 이동합니다:
   ```bash
   cd "C:\Users\master\Desktop\동행 프로젝트"
   ```
2. 포함된 PowerShell 가벼운 HTTP 서버 스크립트를 실행합니다:
   ```powershell
   powershell -ExecutionPolicy Bypass -File server.ps1
   ```
3. 브라우저를 열고 아래 주소로 접속합니다:
   ```
   http://127.0.0.1:8000/
   ```
